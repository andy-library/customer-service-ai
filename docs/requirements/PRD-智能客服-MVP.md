# 产品需求文档（PRD）— 智能客服系统 MVP

| 属性 | 值 |
|------|-----|
| 版本 | 1.0 |
| 日期 | 2026-07-15 |
| 产品名称 | 企业智能客服（Customer Service AI） |
| 阶段 | MVP |

---

## 1. 问题陈述

企业客服存在：

- 多模型能力分散，无法按问题类型自动选择合适模型。
- 业务知识沉淀在文档中，通用大模型无法稳定引用，易出现幻觉。
- 缺少统一的技术底座，难以做后续的质检、运营与微服务扩展。

## 2. 目标用户

| 角色 | 诉求 |
|------|------|
| 客服/业务运营 | 快速得到有依据的答复，可追溯知识来源 |
| 知识管理员 | 上传/维护企业文档，形成知识库 |
| 研发/平台 | 可配置多模型、可观测路由、可本地部署验收 |

## 3. 业务目标

1. 用户提问后，系统自动选择模型并（在需要时）结合知识库回答。  
2. 回答可展示意图、所用模型与知识引用，便于运营与排障。  
3. 形成可演进的模块化技术底座（后续可拆微服务）。

## 4. 范围

### 4.1 In Scope（MVP）

| ID | 需求 | 优先级 |
|----|------|--------|
| R1 | 对接 ≥2 个 OpenAI-compatible 模型（配置化） | P0 |
| R2 | LLM 意图分类路由到回答模型 | P0 |
| R3 | 文档上传（PDF/MD/TXT）并向量化入库 | P0 |
| R4 | 问答时 RAG 检索并返回 sources | P0 |
| R5 | 同步问答 API | P0 |
| R6 | SSE 流式问答 API | P1 |
| R7 | 简易管理后台（知识/对话测试/模型查看） | P0 |
| R8 | 会话 ID 与最近多轮上下文 | P1 |
| R9 | 路由决策日志 | P1 |
| R10 | Docker Compose 一键启动与验收文档 | P0 |

### 4.2 Out of Scope

- 微服务拆分、注册中心、API 网关集群  
- 登录鉴权 / SSO / 细粒度权限  
- 工单、转人工、渠道接入（微信/企微等）  
- 多租户、计费、内容安全审核平台  
- 语音/图像多模态  

## 5. 用户故事

### US-1 智能问答

**作为** 业务用户，**我希望** 用自然语言提问并得到答案，**以便** 快速解决业务问题。

**验收：**

- Given 系统已配置模型，When 发送有效问题，Then 返回非空 answer 与 route 信息。  
- When 问题属于产品类，Then intent 为 PRODUCT（或合理近似）且 answerModelId 为映射配置。

### US-2 知识增强回答

**作为** 业务用户，**我希望** 答案基于企业知识库，**以便** 信任答复并核对来源。

**验收：**

- Given 已上传含「退款规则」的文档，When 问退款规则，Then sources 非空且 answer 引用相关内容。  
- Given 知识库无相关内容，When 提问冷门内部政策，Then 回答声明依据不足，不编造具体条款编号。

### US-3 知识管理

**作为** 知识管理员，**我希望** 上传并管理文档，**以便** 持续沉淀企业知识。

**验收：**

- 可上传 PDF/MD/TXT，状态变为 INDEXED。  
- 可列表、删除；删除后检索不再命中。

### US-4 模型与路由可观测

**作为** 平台研发，**我希望** 看到每次问答用了哪个意图和模型，**以便** 调优路由策略。

**验收：**

- API 响应含 route 对象。  
- 管理台或 API 可查看模型列表与意图映射。

### US-5 对话测试台

**作为** 运营/研发，**我希望** 在网页上测试问答，**以便** 无需 Postman 即可验收。

**验收：**

- `/admin/chat` 可发送问题并展示 answer、route、sources。

## 6. 功能需求明细

### 6.1 模型网关

- 支持配置多个模型：id、baseUrl、apiKey、modelName、role、enabled、timeout。  
- role 至少支持：CLASSIFIER、ANSWER、EMBEDDING（embedding 可走独立配置块）。  
- 对外不暴露 apiKey。  
- 调用失败有明确错误码与日志。

### 6.2 智能路由

- 使用 CLASSIFIER 模型对用户问题分类。  
- 意图集合：PRODUCT、BILLING、TECH_SUPPORT、POLICY、CHITCHAT、UNKNOWN。  
- 按配置映射到 answer 模型。  
- 分类失败降级为 UNKNOWN + 默认模型。

### 6.3 知识库与 RAG

- 支持文件类型：`.pdf`、`.md`、`.txt`。  
- 单文件大小上限：10MB（可配置）。  
- 切片与 topK、阈值可配置。  
- 回答附带 sources：documentId、title、snippet、score。

### 6.4 对话

- `sessionId` 可选；支持最近 N 轮（默认 6 条消息）。  
- 同步与流式两种接口。  
- 可选 `overrideAnswerModelId`、`forceRag` 便于调试。

### 6.5 管理后台

- 知识库管理页  
- 对话测试页  
- 模型/路由只读页  

## 7. 非功能需求

| 类型 | 要求 |
|------|------|
| 运行时 | OpenJDK 21 |
| 框架 | Spring Boot 3.3.x；Spring Cloud BOM 2023.0.x（不强制拆服务） |
| AI | Spring AI 1.1.x（与 Boot 3 兼容的最新稳定线） |
| 性能（本地） | 同步接口在模型正常时 P95 &lt; 15s（不含冷启动，依赖外部模型 SLA） |
| 可用性 | 分类失败可降级，不导致进程崩溃 |
| 安全 | 密钥环境变量注入；上传类型与大小限制 |
| 可维护 | 模块分包清晰；关键配置外置 |

## 8. 依赖与假设

- 用户可提供至少两个 OpenAI-compatible 端点（或开发使用 mock profile）。  
- Embedding 模型维度与库表向量维度一致。  
- 演示环境可访问 PostgreSQL（Compose 提供）。
